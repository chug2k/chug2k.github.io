TL;DR

How Ramp actually operates

  1. Ramp shipped 500+ features and passed $1B in revenue with ~25 PMs by going aggressively AI-native: ~50% of its code is AI-written (up from 30% in December, headed for 80%).
  2. The workflow flipped — PMs no longer write specs for engineers; the spec is a prompt and the output is a working product. PMs, designers, even sales ship real PRs via Ramp's internal 'Inspect' tool, a chunk auto-approved.
  3. Agents run the unglamorous work: a Voice-of-Customer agent sifts Gong calls, tickets, and Salesforce to surface problems (8 minutes vs 8 days); Ramp Research answers any data question (Snowflake CLI + Claude + custom skills).
  4. The leader's job shifts from 'best at the craft' to fixing the process: when output is wrong, ask which prompt/skill/design-system failed and fix that, so the feedback never recurs. Planning horizon: ~3 months.
  5. The PM role isn't dead but splits — become a builder (code is free) or own the business/GM side (strategy, positioning, monetization). 'Management is probably dead'; software becomes 'coworkers,' and the bar is being the best builder.

01 · The numbers

What AI-native actually looks like

Ramp is one of the fastest-growing companies ever and, Peter argues, the most AI-native outside the big labs: 500+ features shipped last year, past $1B in revenue, with only ~25 PMs.

50% of Ramp's code is built by AI. That's 50% up from 30% in December. It'll probably be 80% by March.— Jeff, CPO of Ramp

02 · Prompt → product

The spec is for the AI now

PMs used to pride themselves on the perfect spec. But the spec is now read by an AI, not an engineer — so it's just the output of a prompt, and its output is the product.

We don't really talk about the spec itself — that's just a step. We talk about the actual product. It's prompt to product, back to prompt, back to product.— Jeff

03 · The Voice-of-Customer agent

8 minutes, not 8 days

The hardest part is sifting signal from 50,000+ customers. Ramp's Voice-of-Customer agent reads Gong recordings, Salesforce notes, in-app surveys, support tickets, chats, and Snowflake — and answers any PM question about a persona's pain points.

04 · Ask your data anything

Ramp Research

Making data easy to query made more people query it — so the company got more data-centric. "Ramp Research" understands the whole schema and answers questions in minutes; it's since moved to Snowflake CLI + Claude + custom skills.

The mindset shiftDon't just ask the question — tell the AI your goal. "You'll be surprised at the questions the AI asks itself to get there."

05 · Everyone ships

The Inspect tool

Ramp built "Inspect," a visual layer over LLMs so non-engineers can build real product — front-end and back-end, with the design component library baked in.

Feedback closes the loop automatically: an escalation or support ticket → AI takes the first pass, finds the code, opens the PR, often ships it.

06 · The leader's new job

Fix the process, not the output

When Jeff sees a bad result, the high-leverage question isn't "how do I fix this?" — it's what broke down? Which prompt, skill, or design-system failed? Fix that, and the feedback never recurs.

Planning, meanwhile, runs on a ~3-month horizon ("3 months is what used to be 3 years") — mostly aligning on strategy and trade-offs, not a roadmap.

07 · Is the PM dead?

No — it splits in two

Jeff's surprising take: automating code is "over for the engineer" more than the PM. But the old PM training — stakeholder management, prioritization, frameworks — is outdated. The role splits two ways.

  1. L0occasional ChatGPT user — "most likely won't be at the company."
  2. L1builds custom GPTs / Notion agents.
  3. L2built an app that automates part of their job; commits code.
  4. L3a systems builder who lifts everyone else up.

Ramp removes token/budget limits, runs public "what I built" channels and office hours, requires AI proficiency to be hired (a build-a-product interview), and tracks per-person AI usage.

08 · Software is dead

Long live coworkers

Token cost is a non-issue ("consumption per employee isn't close to double-digit % of salary — if an agent does 10× your work, why wouldn't you pay it more than you?"). The real race is using the window before AI one-shots a whole platform.

Management is probably dead… now is the time to be very, very proficient in this new technology. Optimize to be the best builder in the world.— Jeff
